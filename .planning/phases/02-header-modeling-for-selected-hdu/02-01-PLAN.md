<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI argument → CFITSIO file open | Untrusted filesystem path and selector text cross into library parsing |
| FITS header bytes → in-memory model | Untrusted FITS header content is transformed into owned heap data |
| In-memory model → stdout JSON | Structured internal state crosses into user-consumed JSON output |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | T | `src/fits2json.c` header normalization | mitigate | Use CFITSIO parsing/classification APIs (`fits_read_keyn`, `fits_get_keyclass`, `fits_read_key_longstr`) rather than custom 80-char card parsing; reject/propagate non-zero status immediately. |
| T-02-02 | D | long-string handling and heap allocations | mitigate | Centralize long-string reads, free CFITSIO-owned buffers with `fits_free_memory`, free all model allocations on error paths, and cover long-string/HIERARCH with the synthetic regression fixture. |
| T-02-03 | I | stdout JSON emission | mitigate | Build the whole selected-HDU model first, close the FITS handle before emitting stdout, and emit JSON in one pass so failures cannot leave partial/malformed output. |
</threat_model>
