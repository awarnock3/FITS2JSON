<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI argument → CFITSIO open/select | Untrusted file path and selector syntax enter FITS parsing |
| FITS header strings → JSON emitter | File-controlled text crosses into shell-consumed JSON |
| In-memory whole-file document → stdout | Final document must remain atomic and pipe-safe |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | T | `src/fits2json.c` selectorless traversal/emission path | mitigate | Delegate selector parsing and FITS structure handling to CFITSIO, check status after every FITS call, and only emit selectorless JSON after full traversal plus `fits_close_file` succeed. |
| T-03-02 | T | JSON serialization of FITS keyword/value/comment strings | mitigate | Reuse `write_json_string` for every emitted string field in both single-object and array emitters; never print raw FITS card text directly. |
| T-03-03 | D | Whole-file in-memory document assembly | accept | Phase 3 intentionally builds the document in memory to avoid partial stdout. Keep allocation-failure cleanup paths and return an error with no stdout output; local CLI scope makes this an acceptable residual risk for very large headers. |
</threat_model>
